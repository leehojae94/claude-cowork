# 차량정보조회 API 테스트 도구 — 배포 가이드

## 파일 구조

```
car-api-tester/
├── index.html       ← 화면 (사용자가 보는 페이지)
├── api/
│   └── proxy.js     ← 프록시 서버 (API 토큰 보관)
├── package.json     ← 프로젝트 설정
├── vercel.json      ← Vercel 배포 설정
└── 배포가이드.md
```

---

## STEP 1 — GitHub 저장소 만들기

1. https://github.com 접속 → 로그인
2. 우측 상단 `+` 버튼 → **New repository** 클릭
3. Repository name: `car-api-tester` 입력
4. **Private** 선택 (내부용이므로)
5. **Create repository** 클릭
6. 생성된 저장소에 이 폴더 안의 파일 4개 업로드
   - `index.html`
   - `api/proxy.js` (api 폴더 안에)
   - `package.json`
   - `vercel.json`

---

## STEP 2 — Vercel 연동

1. https://vercel.com 접속 → **GitHub으로 로그인**
2. **Add New → Project** 클릭
3. GitHub 저장소 목록에서 `car-api-tester` 선택 → **Import**
4. 설정 화면에서 아무것도 바꾸지 않고 **Deploy** 클릭
5. 배포 완료 후 공개 URL 발급 (예: `https://car-api-tester.vercel.app`)

---

## STEP 3 — API 토큰 환경변수 설정 ⚠️ 필수

> 이 단계를 빠뜨리면 API 조회가 동작하지 않습니다.

1. Vercel 프로젝트 대시보드 → **Settings** 탭
2. 좌측 메뉴 **Environment Variables** 클릭
3. 아래 값 입력 후 **Save**

| 항목  | 값                                         |
|-------|--------------------------------------------|
| Name  | `API_TOKEN`                                |
| Value | `94c7362ab2c849b686b84c3863f6f791efeb2850` |

4. **Deployments** 탭 → 최신 배포 우측 `⋯` → **Redeploy** 클릭
   (환경변수는 재배포 후 적용됩니다)

---

## STEP 4 — 접속 테스트

- 발급된 URL(예: `https://car-api-tester.vercel.app`)로 접속
- 자동차등록번호 + 소유자명 입력 후 조회하기 클릭
- 팀원에게 URL 공유하면 누구나 사용 가능

---

## 토큰 변경 시

Vercel → Settings → Environment Variables에서 값 수정 후 재배포
